# Component Recipes

Esta pasta documenta como os tokens publicos devem ser compostos em componentes reais.

Cada recipe:

- vive fora de `tokens/` para nao virar foundation do design system
- referencia apenas paths publicos do contrato semantico
- descreve slots, tamanhos, estados e motion de forma consumivel por frontend

Arquivos incluidos:

- `button.json`
- `input.json`
- `card.json`
- `modal.json`
- `badge.json`
- `toast.json`
